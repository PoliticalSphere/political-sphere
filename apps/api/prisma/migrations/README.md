# Prisma Migrations

This directory contains database migration files generated by Prisma Migrate.

⚠️ **DO NOT EDIT MIGRATION FILES MANUALLY**

Migration files are auto-generated when you run:

```bash
npx prisma migrate dev --name description_of_change
```

## Migration Workflow

### Development

1. **Modify schema**: Edit `schema.prisma` with your changes
2. **Create migration**: Run `npx prisma migrate dev --name add_user_table`
3. **Review migration**: Check the generated SQL in the new migration folder
4. **Apply migration**: The migration is automatically applied in dev mode

### Production

1. **Deploy migration**: Run `npx prisma migrate deploy` in production
2. **Verify**: Check that the migration was applied successfully

## Migration Files

Each migration creates a folder with:

- `migration.sql` - The SQL statements to execute
- Timestamp-based folder name (e.g., `20250108120000_add_user_table/`)

## Troubleshooting

- **Reset database**: `npx prisma migrate reset` (⚠️ destroys all data)
- **Resolve conflicts**: `npx prisma migrate resolve`
- **View status**: `npx prisma migrate status`

## Best Practices

- Use descriptive migration names
- Review generated SQL before committing
- Never modify migration files after they're applied
- Keep migrations small and focused
- Test migrations on staging before production
